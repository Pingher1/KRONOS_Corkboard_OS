## BOOK I: ARCHITECTURAL GENESIS (THE MODERN CONFIG MATRIX)
What used to take just two files in the MS-DOS era (`CONFIG.SYS` and `AUTOEXEC.BAT`) has structurally evolved into the "Modern Six" for a Sovereign Electron/React Operating System. The raw order of operations logic is identical, it is simply separated into specialized handlers:

1. **`package.json` (The Main OS Registry)**
   - Acts identically to `CONFIG.SYS`. It explicitly details every single application dependency, version constraint, and the exact commands to start the machine (`npm run electron:dev`).
2. **`electron/main.cjs` (The OS Bootloader)**
   - The modern equivalent of `AUTOEXEC.BAT`. This file bypasses the Chrome browser environment and physically summons the isolated, CORS-bypassing desktop GUI window natively onto your macOS framework.
3. **`index.html` (The Physical Canvas)**
   - The absolute bedrock graphical payload. It simply provides the `<div id="root"></div>` void for the React Matrix to render upon.
4. **`vite.config.js` (The Translation Engine)**
   - The rapid-hot-reload engine that compiles raw ES6 JavaScript and Tailwind CSS into hyper-optimized machine instructions instantly as we program.
5. **`tailwind.config.js` (The Physics Token Registry)**
   - The CSS master dictionary. It regulates how the Apple Liquid Borders, 3D CSS `translateZ` grids, and absolute sizing modules function across all `.jsx` files.
6. **`src/App.jsx` (The Master Logic Controller)**
   - Phase II Execution. Where the actual KRONOS terminal components live: the Slot Machine Wheel, the 4.5 Piggyback Engine, the Daylight Inversion toggle, and the Northstar/Southstar Ribbons.
